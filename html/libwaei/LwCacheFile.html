Description

This is a mapped file designed two have two modes, making it ideal for handling data cached by the application that is read many times, but only written once. When lw_cachefile_write() or lw_cachefile_write_cachefile() is called, the LwCacheFile is set to writing mode, where the checksum is written first, then any contents are written after. Consecutive calls append to this file. Once you call lw_cachefile_read(), the FILE steam is closed if it is open and the file is verified as matching the supplied checksum in the header and that the contents are valid utf8.
