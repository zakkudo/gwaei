Description

This class forms a smaller part of LwParsed, which represents a whole parsed file. You will not usually be creating/destroying LwParsedLines directly.

LwParsedLine is designed to minimize memory allocations by being able to be declared on the stack, use the binary data it was deserialized from to hold it's strv pointer array, and having the strvs reference strings in a separate chunk. This means there are a lot of gotchas if you try to use this class outside of the LwParsed context.

Usage example:

LwParsedLine parsed_line = {0};
lw_parsedline_init_full (&parsed_line, (GDestroyNotify) g_strfreev);

lw_parsedline_set_strv (&parsed_line, 0, g_strsplit (":", "one:two"));
printf("%s\n", lw_parsedline_get_strv (&parsed_line, 0)[0]);

lw_parsedline_clear (&parsed_line);

lw_parsedline_serialize ()

gsize
lw_parsedline_serialize (LwParsedLine *self,
                         gchar const *contents_reference_point,
                         gchar *preallocated_buffer,
                         GError **error);

Serializes a LwParsedLine such that it may be written to a file and read at a later date. The Strings are converted from pointers to relative byte offsets, making contents_reference_pointer required to preform the calculations. Alls strvs recorded in the LwParsedLine must be pointing to the same chunk of memory as their base for serialization and deserialization to work.

lw_parsedline_deserialize_into ()

gsize
lw_parsedline_deserialize_into (LwParsedLine *self,
                                gchar const *serialized_data,
                                gchar const *contents_reference_point,
                                GError **error);

Deserializes the data from serialized_data into the LwParsedLine object. It does this by taking the string offsets and converting them back into pointers with contents_reference_point as the base. As a sanity check the strings all are validated as utf-8 and if they fail, an error is set and returned. Because serialized_data is used to hold the string array data, it is and error deserialize serialized_data twice and it will return an error. For the same reason, is also an error to free serialized_data after deserialization.

Parameters

self

A LwParsedLine that should be initialized to {0}

  

serialized_data

The data to build the LwParsedLine from

  

contents_reference_point

File used as the base offset to restore the strvs

  

error

An GError to record errors or NULL

  

Returns

The bytes read from serialized_data to build the LwParsedLine
